# Definition of Done

Ein Backlog-Item ist „Done“, wenn:
1. Fachliche Akzeptanzkriterien erfüllt sind.
2. Code gebaut und automatisierte Tests grün sind.
3. Relevante Unit-/API-Tests vorhanden sind.
4. Öffentliche APIs und zentrale Domänentypen dokumentiert sind.
5. Logging/Fehlerbehandlung für Betriebsanalyse ausreicht.
6. Sicherheits- und Datenschutzaspekte geprüft wurden.
7. Änderungen in Doku/ADR nachvollziehbar ergänzt wurden.
8. PR-Review abgeschlossen und Merge-Strategie eingehalten ist.
